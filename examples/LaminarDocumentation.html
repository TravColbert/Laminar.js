Laminar

A simple, programmatic web-UI toolkit.

Get Laminar See The Examples

Basics

Laminar objects are created programatically.

Here is a basic Laminar object. By default, Laminar objects are DIVs but they can be any DOM object type by using the element property.

Notice that the ID of the object is firstelement while the Javscript name for this Laminar object is firstElement (note the camel-case lettering). Knowing the Javascript name of the object allows you to programmatically modify the object.

  firstElement = new Laminar.Widget({
    id:"firstelement",
    content:"My first Laminar widget",
    classlist:["demo"],
    parent:".content"
  }).update();

Result:

If you look at the source of this page you will see that all of the elements we are creating have a class of demo. This is so that we can add a little bit of styling to our created Laminar objects to make them easier to see. By default, Laminar objects do not come with the demo class attached.

You can see that we define a Laminar widget by giving the Laminar.Widget constructor a configuration object that looks like this:

  {
    id:"firstelement",
    content:"Some text",
    classlist:["demo"],
    parent:".content"
  }

After passing-in the configuration object to the constructor we call the widget's update() method which places the widget in the DOM.

This made an element in this web page (in the element with the content class that looks like this behind the scenes:

  <div id="firstelement" class="lm demo">Some text</div>

You see where all of the elements in the DOM object come from. The only one that we didn't specify was the class attribute of lm. Every Laminar object gets the lm class.

Parent

The parent property (above) says .content. This property specifies that the resulting Laminar object will be placed in the element with the class of content in the rendered web page.

You may specify any pre-existing DOM element as a parent for your Laminar object - as long as the element is a container type object. If you don't specify a parent property when you build your Laminar object, the object will be created, but will not be placed in a DOM element in your web page.

This isn't a bad thing. It allows you to build Laminar objects, manipulate them, then finally place them in your web page later.

Changing Parent

You can use the method: setParent() to change a Laminar element's parent from one DOM container to another. This is the same as moving a DOM element.

Try using the setParent() method here.

Contents

You see that property called content in the code above? The content property lets you fill your Laminar object with something else - text, HTML, even another Laminar object!

Let's add some more elements inside our Laminar object:

  childObject = new Laminar.Widget({
    id:"cobject",
    element:"input",
    content:"I am the child!",
  }).update();

  parentObject = new Laminar.Widget({
    id:"pobject",
    content:["I am the parent!", childObject],
    parent:".content"
  }).update();

When we want to add multiple items as contents of an object, simply package them in an array.

Elements

Laminar objects don't have to be just DIVs.

Let's create a simple paragraph ("p") element. Use the element property to specify what you want to make:

  secondElement = new Laminar.Widget({
    id:"paragraphelement",
    element:"p",
    contents:"This is a paragraph that was created programatically"
  }).update();

See how we added the "element" property to specify what kind of DOM element we were making?

How about a button? Easy:

This button doesn't do anything yet, but at least it is there. Later we will learn how to make our Laminar objects do something.

We created this Laminar object like this:

  buttonElement = new Laminar.Widget({
    id:"linkelement",
    element:"input",
    type:"button",
    value:"Press Me",
    content:"This is a button",
    parent: "#buttonholder"
  }).update();

See how we have added a type and value property to our Laminar object? Also, note that we told the button that its parent container was #buttonholder. Parents can be any valid CSS selector. Naturally, it should return just one DOM container object.

Simply changing the type to "textinput" makes the object render like this:

By the way... how are we making the above element switch back and forth between "textinput" and "button"?

Simply by telling the Laminar object we created earlier called buttonElement to set its "type" to another type of input. We do it in plain Javascript by referencing the Javascript object's name like this:

  buttonElement.set("type","textinput");

We simply called the Laminar widget's set() method to change the type setting to something else.

There's no need to add any fancy add-on libraries to manipulate Laminar objects. It's all built-in to each one of them.

Pre-existing Elements

We've created brand-new DOM elements using Laminar. However, we can also turn normal DOM elements into powerful Laminar objects!

Let's turn this garden-variety button...

  <div id="custombuttonholder" class="demo">
    <input type="button" id="abutton" value="Typical Button">
  </div>

...into a Laminar button.

We do this by referencing the pre-existing DOM object when we create the Laminar object. Like this:

  var customLaminarButton = new Laminar.Widget(document.getElementById("abutton")).update();

Notice how we simply gave the Laminar widget constructor the DOM element? We did that by using HTML5's getElementById() function. Laminar 'gobbles' up this DOM object and turns it into a Laminar object with all of Laminar's abilities baked-in.

The result is a Laminar object that we can refer to by the name: customLaminarButton. If you open your browser's developer tools (by pressing F12, Ctrl-I or something like that) you can programmatically modify this little button. Try changing the background color of the button like this:

  customLaminarButton.css("background-color","lightgreen");

You will probably even notice that as you type the new Lamainar button your browser will start to complete the typing for you. It may even list the methods that bundled into your new Laminar button.

Don't worry. Your shiny new Laminar object will still contain all of its original child elements:

Classes

Now let us try creating an object and giving it a class.

Let's make the class: "urgent":

  thirdElement = new Laminar.Widget({
    id:"divelement",
    content:"This is urgent text",
    classlist:["urgent"],
    parent:"#holderdiv3"
  }).update();

You can specify as many classes as you want:

  fourthElement = new Laminar.Widget({
    id:"statusmessage",
    content:"Here is an important message",
    classlist:["status","caption"],
    parent:".content"
  });

We can toggle classes for objects that have already been made:

  fifthElement.toggleClass("status");

CSS

We can also programatically manipulate CSS styles directly:

  fifthElement.css("color","#e94609");

Events

You can tell an Laminar object what to do when it senses an event. We use the listenEvent() method:

  LaminarObject.listenEvent(event,function);

The listenEvent() method expects an event, and a function that is called when the event is fired.

The function should be written to accept two parameters: the event object (commonly noted as 'e'), and the Laminar object that caught the event. Take the example below. In this example, we decided to get a little evasive by adding a function called trigger() that is called when a "click" event is caught by the Laminar button widget we're going to create:

  var trigger = function(e, obj) {
    obj.toggleClass("right");
  }

  shiftyElement.listenEvent("click",trigger);

See how we created a function, then we told our Laminar object to execute that function when it is clicked?

Try clicking it:

The "trigger" function simply toggles a class that positions the element. Every time you click the element, the trigger function is fired and the class is toggled on and off.

Other Event Methods

Sometimes it is good to have a little more control over when the events actually fire... and if they fire at all. You might, for example, have a button or a link that you only want to have fired once. Other Javascript libraries have a method called once that can be bound to DOM objects. Laminar has the same ability but it is in the listenEvent() method.

Simply pass the listenEvent() method an object with doOnce equaling true.

  singleUseElement = new Laminar.Widget({
    id:"singleuseelement",
    value:"Click Me!",
    element:"input",
    type:"button",
    classlist:["slow"],
    parent:".content"
  }).update();

  shiftyElement.listenEvent("click",trigger,{'doOnce':true});

Dimensions & Position

Sometimes you need to know the dimensions and positioning of your Laminar objects. You can ask any Laminar object where it exists in the universe using getTop(), getLeft(), or getPosition().

You may have noticed that this Laminar object changes when the previous Laminar object (called "shifty") moves. This is another example of setting events to our objects.

  locatorElement = new Laminar.Widget({
    id:"locatorelement",
    content:"shiftyElement is: " + shiftyElement.getLeft() + "(x) " + shiftyElement.getTop() + "(y) " + shiftyElement.getHeight() + "(h) " + shiftyElement.getWidth() + "(w)",
    classlist:["demo"],
    parent:".content"
  }).update();

  var updateSize = function(e, obj) {
    locatorElement.empty();
    locatorElement.content("shiftyElement is: " + obj.getLeft() + "(x) " + obj.getTop() + "(y) " + obj.getHeight() + "(h) " + obj.getWidth() + "(w)")
  }

  shiftyElement.listenEvent("transitionend",updateSize);

You might use this ability to implement responsiveness right into the Laminar objects themselves. Let's make an element that recomposes itself based on its width.

The above Laminar box will react as you resize the window.

The key to how this widget works is the new configuration property: responsive.

The responsive property creates an event-listener behind the scenes that listens for resize events on the DOM. When a resize event happens, the supplied function is invoked.

Here is how the above example looks:

  var longText = 'This is a DIV that responds to changes in page width. This element will change its contents depending on what its width is'
  var checkWidth = function(thisObj) {
    var dimension = thisObj.getDimensions();
    if(dimension.width<200) {
      thisObj.setState('wtiny');
      thisObj.html("DIV");
    } else if(dimension.width<400) {
      thisObj.setState('wsmall');
      thisObj.html("Responsive DIV");
    } else if(dimension.width<600) {
      thisObj.setState('wmed');
      thisObj.html('This is a Responsive DIV');
    } else {
      thisObj.setState('wfull');
      thisObj.html(longText);
    }
  };
  var responsiveDiv = new Laminar.Widget({
    id:'responsivediv',
    content:longText,
    classlist:["demo","responsive"],
    statelist:["wfull","wmed","wsmall","wtiny"],
    parent:".content",
    responsive:checkWidth
  }).update(checkWidth);

So, let's break down what is happening here.

The first line simply creates the text that will appear in our Laminar DIV object.

The second statement creates a function called checkWidth() that does two things depending on the width of the object that is passed to it:

  • Sets the STATE of our Laminar DIV to one of 4 possible classes: wfull, wmed, wsmall, wtiny
  • Sets the text that should appear in the DIV

Finally the last statement creates the Laminar DIV. It is defined as a state-type object so that it can only ever be in one of only 4 states.

Most importantly, the responsive property is set to the function defined above. Every time the Laminar DIV object detects a resize in the web page, it will invoke the responsive function which adds exactly one of 4 classes to the object.

Or course, CSS rules defined elsewhere change the color of the DIV based on the class applied to the element.

HTML5 Data Attributes

HTML5 introduced data attributes. These were a nice provision to store extra data in an element in a standard way.

Laminar objects work well with these data attributes. You can either define data attributes or programatically assign data attributes. Here's how:

  oneAttributeElement = new Laminar.Widget({
    id:"attributeelement",
    content:"This element has data",
    classlist:["demo","other","classes"],
    parent:".content",
    datalist:{location:"New York",population:"8,406,000"}
  }).update();

You see how we passed a JSON object to Laminar in the datalist attribute? That object should contain the keys and values for the data that you wish to embed in your DOM element.

The result looks like this:

Of course, what really matters is what the HTML looks like:

  <div id="attributeelement" class="lm demo" data-location="New York" data-population="8,406,000">This element has data</div>

Laminar adds the 'data-' prefix to your datalist keys to make them legal data attributes in HTML5.

You're also free to add data attributes programmatically by using the setData() method of the Laminar widget:

  oneAttributeElement.setData({nickname:"The Big Apple"});

The new data attribute will be added to the Laminar object.

Naturally, Laminar provides an easy, programatic way of reading data attributes too. Simply use the getData() method. Pass the attribute name (minus the 'data' part) to the getData() method of the Laminar object to see what it is set to.

State Objects

Related to classes are states. This is a little different from classes. Any class may be arbitrarily added and removed. A 'state object', on the other hand, wants to always be in at least one of a set of possible states.

Maybe you are designing some cool, custom 3-way toggle control. The toggle control you are designing must always be in one of only 3 states. It can never not be in one of three states.

Laminar makes it easy to implement such a thing. Take a look:

State objects build upon base objects by adding methods that control how the object moves from one state to another.

Here, we incrementally loop through all of the possible states of this Laminar object. We've defined what all the possible states of this object are like this:

  stateElement = new Laminar.Widget({
    id:"astateelement",
    content:"I am a state object",
    element:"input",
    type:"button",
    statelist:["on","off","idle"],
    parent:"#holderdiv8"
  }).update();

You see that we gave 'stateElement' a list of all of the states that it can possibly be in ("on", "off", "idle"). The property that defines these possible states is called statelist.

Notice that we've also not given this object a value (buttons usually display thier 'value' attribute). Instead, we set the value of this object after it has been created by using the set() method - setting the 'value' attribute to whatever the current state of the object is.

Finally, we've created an anonymous function that sets the state of the object to whatever the next state should be when the button is clicked. Then, it sets the value of the object to whatever the current state is, like this:

  // Make the button show what the current state is
  stateElement.set("value",stateElement.getState());

  // When clicked, invoke this anonymous function which
  // 1. Sets the next state
  // 2. Sets the 'value' to the new state thereby showing the state on the button face
  stateElement.listenEvent("click",function(e, obj) {
    obj.setNextState();
    obj.set("value",obj.getState());
  })

If you try to set a Laminar state object to a non-existent state, the object will simply refuse:

  stateElement.setState("bad_state");
  // Nothing happens

  stateElement.setState("idle");
  // Button state set to idle
